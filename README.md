<h2>ОБЩЕЕ ОПИСАНИЕ ЗАДАНИЯ</h2>
Создать серверную часть приложения магазина "ShopOnline" на Node.js (JavaScript).<br />
База данных для хранения данных о товарах - PostgreSQL.<br />
Сервер должен предоставлять API для работы с товарами, категориями и расчетом общей
стоимости товаров.<br />
API должно поддерживать операции CRUD (создание, чтение, обновление и удаление) для товаров и
категорий.<br />
Сервер должен предоставлять возможность получить список дисконтных товаров и общую
стоимость всех товаров.<br />
Предусмотреть возможность загрузки изображений товаров в формате base64 и сохранения их на
сервере в папке ./image.<br />
Обеспечить обработку ошибок при работе с API, включая валидацию данных и возврат
соответствующих статусов HTTP.<br />

<h2>ДЕМОНСТРАЦИЯ РЕШЕНИЯ</h2>
Для демонстрации работы API серевера разработано CMS - ShopOnline.<br /><br />
<b>В гостевом режиме</b> возможен постраничный просмотр списка товаров, поиск по наименованию, фильтрация по категории или дисконтных товаров.<br />
Реализован расчет итоговой суммы товаров с учетом фильтров.<br />
Предусмотрен просмотр фото и информации о товаре.<br />
<br>
<b>В режиме администратора</b> дополнительно возможен отбор по статусу товара: <i>"все товары"</i> или <i>"продажа"</i> или <i>"оформление"</i>.<br />
Товары со статусом <i>"оформление"</i> не видны в гостевом режиме.<br />
Доступно добавление, редактирование и удаление товара. Загрузка фото товара.<br />

<h2>УСТАНОВКА И ЗАПУСК CEРВЕРА</h2>

1. Установите на компьютере c официальнога сайта Node.js версии 22.22.2
2. Скачайте `shoponline-main.zip` (кнопка `<>Code` пункт `Download ZIP`) и распакуйте в папку `shoponline`
3. Скопируйте файл `.env` (передан куратору) в папку `shoponline`
4. В папке `shoponline` запустите команту `npm i`
5. Запуск сервера командой `node entry.cjs`
   <br>
   <b>Примечание</b><br />
   <i>Сервер и база данных PostgreSQL размещены на Спринтхост.</i>
   <i>Так как Passenger на Спринтхосте не умеет загружать ES-модули, <br> введен дополнительный файл `entry.cjs` для загрузки основного `api.mjs`</i>

<h2>База данных для тестирования на localhost</h2>

1. В pgAdmin для сервера PostgreSQL v.18 (или выше) создайте базу `shoponline`
2. Восстановите базу из копии `.\PostgreSQL 18\shoponline.sql`
3. Создайте файл `.env`

Пример файла **.env**

```js
PORT=3000
DB_HOST = localhost
DB_PORT = 5432
DB_NAME = shoponline
DB_USER = postgres
DB_PASSWORD = admin
DIR_LOGS = ./error_logs
DIR_IMAGES = ./public/image
PREFIX_NAME_IMAGE = image/
DEF_NAME_IMAGE = image/prototype.jpg
```

<h3>Формат fetch-запросов:</h3>

```js
await fetch(
  'httpAPI / маршрут ? параметр1=значение & параметрI=значение & autch=hashПароля',
  {
    method: 'метод',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  },
);
```
